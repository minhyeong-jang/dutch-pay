# 나란(naran.im) 랜딩 페이지 디자인 가이드

## 1. 외부 리서치 요약

### 1.1 경쟁사 랜딩 페이지 분석

#### Splitwise (splitwise.com)
- **구조**: Hero(가치 제안) → 핵심 기능 4가지 → PRO 기능 → 기능 비교표 → 미디어 인용/사회적 증명 → Footer
- **스타일**: 깔끔한 미니멀리즘, 카드 기반 구조, 교대 배치 텍스트/이미지
- **CTA**: "Sign up" 반복 배치, 앱 스토어 다운로드 보조 CTA
- **배울 점**: 기능별 명확한 섹션 분리, 미디어 인용을 통한 신뢰도 확보
- **차별화**: Splitwise는 개인 간 1:1 빚 추적 중심 → 나란은 "복수 결제건 교차 정산" 특화

#### Tricount (tricount.com / bunq)
- **구조**: 다크 네이비/블랙 배경 + 밝은 블루(#377ef7) 악센트
- **스타일**: 그라데이션 배경, Inter + Montserrat 폰트, 고정 네비게이션
- **배울 점**: 다크 모드 기반 프리미엄 핀테크 느낌, 색상 대비로 CTA 강조
- **차별화**: 유럽 중심 다국어 → 나란은 한국 사용자 특화 UX

#### Toss (toss.im) - 핵심 레퍼런스
- **구조**: Hero(아이폰 목업) → 기능별 섹션(송금/대출/투자/결제) → B2B → Footer
- **스타일**: 화이트 배경, Toss Blue(#3182f6), Toss Product Sans 전용 서체
- **컬러**: 화이트 + 라이트그레이(#f9fafb) 배경, 다크네이비(#191f28) 텍스트, 블루 악센트
- **타이포**: 50px→28px 반응형 헤드라인, 명확한 위계
- **카피**: "간편하게", "한눈에", "1분 만에" 등 시간/편의 강조
- **배울 점**: 한국어 타이포그래피 최적화, 모바일 퍼스트 640px 브레이크포인트, 깔끔한 여백 활용
- **차별화**: 토스는 종합 금융 플랫폼 → 나란은 정산 한 가지에 집중하는 경량 도구

### 1.2 2025-2026 랜딩 페이지 트렌드

| 트렌드 | 적용 방안 |
|--------|----------|
| **모바일 퍼스트** | 모바일 뷰 우선 설계, sticky CTA, 터치 타겟 44px 이상 |
| **미니멀 레이아웃** | 넉넉한 여백, 2개 이하 폰트 패밀리, 제한된 컬러 팔레트 |
| **마이크로인터랙션** | 무거운 비디오 대신 hover 효과, 스크롤 트리거 페이드인 |
| **콘텐츠 퍼스트** | 명확한 시각 위계, 스캐너블한 카피, 불필요한 장식 배제 |
| **소프트 그라데이션** | 볼드한 컬러 블록 대신 미묘한 그라데이션으로 깊이감 |
| **사회적 증명** | 실사용 후기, 수치 강조 (N명 사용 중) |
| **성능 최적화** | 가벼운 이미지, 지연 로딩, 빠른 FCP/LCP |

## 2. 기존 디자인 시스템 분석

### 2.1 컬러 토큰 (OKLCH 기반)

```
[Light Mode]
--primary:        oklch(0.5955 0.1175 180.72)  → 틸/시안 계열
--primary-fg:     oklch(1 0 0)                  → 화이트
--secondary:      oklch(0.9826 0.0147 173.38)  → 연한 민트
--secondary-fg:   oklch(0.3478 0.0651 178.72)  → 진한 틸
--accent:         oklch(0.7836 0.1647 74.93)   → 앰버/골드
--accent-fg:      oklch(0.2629 0.0579 51.81)   → 다크 앰버
--destructive:    oklch(0.6277 0.2272 27.33)   → 레드
--success:        oklch(0.55 0.1706 163.22)    → 에메랄드
--background:     oklch(0.9823 0 0)            → 거의 화이트
--foreground:     oklch(0.1992 0 0)            → 거의 블랙
--muted:          oklch(0.9703 0 0)            → 라이트 그레이
--muted-fg:       oklch(0.45 0 0)              → 미디엄 그레이
--border:         oklch(0.9222 0 0)            → 연한 그레이

[Dark Mode]
--primary:        oklch(0.7953 0.1467 178.72)  → 밝은 틸
--background:     oklch(0.1336 0 0)            → 거의 블랙
--foreground:     oklch(0.9823 0 0)            → 거의 화이트
```

### 2.2 핵심 디자인 토큰 요약

- **프라이머리 컬러**: 틸/시안 (hue ~180) - 신뢰감 + 신선함
- **악센트 컬러**: 앰버/골드 (hue ~75) - 주의 환기, 포인트
- **서체**: Pretendard Variable (한국어 최적화) + Geist Mono (코드/숫자)
- **모서리 반경**: 0.75rem 기본 (--radius)
- **자간**: -0.01em 기본 (한국어 밀도 최적화)
- **그림자**: 부드러운 spread (2px~10px), 낮은 불투명도
- **컨테이너**: max-width 1200px(lg) / 1400px(xl), 패딩 1rem~2rem

### 2.3 기존 컴포넌트 패턴

- **Button**: 6가지 variant (default/destructive/outline/secondary/ghost/link), 6가지 size
- **Card**: rounded-xl, border, shadow-sm, py-6 내부 패딩
- **Header**: fixed, 높이 14(3.5rem), backdrop-blur, 스크롤 시 border 전환
- **Layout**: container 유틸리티, pb-16(모바일 네비 여백), pt-14(헤더 여백)
- **Touch Target**: min 44x44px 유틸리티 클래스

## 3. 비주얼 방향성

### 3.1 톤앤매너

**키워드**: 깔끔 / 신뢰 / 경쾌 / 실용

- "복잡한 걸 단순하게" → 시각적으로도 군더더기 없는 미니멀
- 한국 20-30대 타겟 → 토스 수준의 현대적 감각, 그러나 더 가볍고 캐주얼
- 핀테크 신뢰감 + 친구 사이 편안함의 균형
- 불필요한 장식 배제, 콘텐츠 중심 레이아웃

### 3.2 무드

| 요소 | 방향 |
|------|------|
| 분위기 | 밝고 개방적, 부담 없는 |
| 색감 | 틸/시안 기반 시원한 톤, 앰버 포인트 |
| 여백 | 넉넉한 섹션 간격 (80px~120px) |
| 이미지 | 추상 일러스트 + 앱 목업 (실제 스크린샷) |
| 카피 톤 | 대화체, 공감형, "~하잖아요" 스타일 |

## 4. 컬러 시스템

### 4.1 기존 테마 유지 (앱과 일관성)

모든 기존 CSS 변수를 그대로 사용합니다.

### 4.2 랜딩 전용 확장 컬러

```css
/* 랜딩 페이지 전용 - 기존 테마 확장 */

/* Hero 배경 그라데이션 */
--landing-hero-from:    oklch(0.97 0.01 180);    /* 연한 민트 */
--landing-hero-to:      oklch(0.98 0 0);          /* 거의 화이트 */

/* 섹션 교대 배경 */
--landing-section-alt:  oklch(0.96 0.005 180);    /* 극미한 민트 틴트 */

/* 강조 그라데이션 (CTA, 배지 등) */
--landing-gradient-from: oklch(0.5955 0.1175 180.72); /* primary */
--landing-gradient-to:   oklch(0.55 0.1706 163.22);   /* success/에메랄드 */

/* 텍스트 하이라이트 */
--landing-highlight:     oklch(0.7836 0.1647 74.93);  /* accent/앰버 */

/* 다크 모드 Hero */
--landing-hero-dark-from: oklch(0.18 0.015 180);
--landing-hero-dark-to:   oklch(0.1336 0 0);
```

### 4.3 컬러 사용 규칙

| 용도 | 컬러 |
|------|------|
| 주요 CTA 버튼 | `primary` (틸) |
| 보조 CTA | `secondary` (연민트) 또는 `outline` |
| 섹션 배경 교대 | `background` ↔ `landing-section-alt` |
| 텍스트 강조 단어 | `primary` 컬러 텍스트 |
| 숫자/데이터 강조 | `accent` (앰버) |
| 성공/완료 상태 | `success` (에메랄드) |
| 경고/주의 | `warning` (앰버) |

## 5. 타이포그래피 가이드

### 5.1 서체

- **본문/제목**: Pretendard Variable (이미 프로젝트에 설정됨)
- **숫자/코드**: Geist Mono (이미 프로젝트에 설정됨)
- **추가 서체 불필요** - 2개 패밀리로 충분

### 5.2 스케일

| 용도 | 모바일 | 데스크톱 | Weight | 자간 |
|------|--------|----------|--------|------|
| Hero 제목 | 32px (text-3xl) | 56px (text-5xl+) | 800 (extrabold) | tracking-tighter |
| Hero 부제목 | 18px (text-lg) | 24px (text-2xl) | 400 (normal) | tracking-normal |
| 섹션 제목 | 24px (text-2xl) | 40px (text-4xl) | 700 (bold) | tracking-tight |
| 섹션 부제목 | 14px (text-sm) | 16px (text-base) | 500 (medium) | tracking-wide |
| 본문 | 16px (text-base) | 18px (text-lg) | 400 (normal) | tracking-normal |
| 캡션/라벨 | 12px (text-xs) | 14px (text-sm) | 500 (medium) | tracking-wide |
| CTA 버튼 | 16px (text-base) | 18px (text-lg) | 600 (semibold) | tracking-normal |
| 숫자 강조 | 28px (text-2xl+) | 48px (text-5xl) | 700 (bold) | tracking-tight, font-mono |

### 5.3 타이포 규칙

- 한국어 `word-break: keep-all` 유지 (이미 body에 적용됨)
- 제목은 2줄 이내, 한 줄 15자 내외 권장
- 섹션 부제목은 primary 컬러 + uppercase/spacing으로 라벨 역할
- 숫자(금액, 시간)는 Geist Mono로 시각적 구분

## 8. 애니메이션/인터랙션 제안

### 8.1 원칙

- **성능 우선**: CSS 애니메이션 + `will-change` 최소 사용, JS 애니메이션 라이브러리 지양
- **미묘함**: 사용자가 의식하지 못할 정도의 자연스러운 전환
- **목적성**: 장식이 아닌 정보 전달을 위한 모션
- **접근성**: `prefers-reduced-motion` 미디어 쿼리 존중

### 8.2 구현 제안

| 요소 | 애니메이션 | 구현 방식 |
|------|-----------|----------|
| **섹션 진입** | 아래에서 페이드인 (translateY: 20px → 0, opacity: 0 → 1) | Intersection Observer + CSS transition |
| **공감 카드** | 순차 등장 (staggered, 100ms 간격) | CSS animation-delay |
| **How it Works** | 단계별 순차 페이드인 | Intersection Observer |
| **특징 카드** | 호버 시 미세 상승 (translateY: -4px) + 그림자 강화 | CSS :hover transition |
| **CTA 버튼** | 호버 시 scale(1.02) + 그림자 확대 | CSS :hover transition |
| **숫자** | 카운트업 애니메이션 (0 → 목표값) | 뷰포트 진입 시 JS |
| **앱 목업** | 미세한 떠다니는 효과 (float) | CSS @keyframes, 3s ease-in-out infinite |
| **네비게이션** | 스크롤 시 배경 블러 전환 | 이미 구현됨 (app-header.tsx) |

### 8.3 CSS 예시

```css
/* 섹션 페이드인 */
.landing-fade-in {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}
.landing-fade-in.visible {
  opacity: 1;
  transform: translateY(0);
}

/* 카드 호버 */
.landing-card {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.landing-card:hover {
  transform: translateY(-4px);
  box-shadow: var(--shadow-lg);
}

/* 접근성 */
@media (prefers-reduced-motion: reduce) {
  .landing-fade-in {
    opacity: 1;
    transform: none;
    transition: none;
  }
}
```

## 9. 아이콘/일러스트 방향

### 9.1 아이콘

- **스타일**: Lucide Icons (이미 프로젝트에서 사용 가능)
- **굵기**: stroke-width 1.5 ~ 2 (기본 유지)
- **크기**: 섹션 아이콘 32px, 인라인 아이콘 16~20px
- **컬러**: primary(틸) 단색, 또는 muted-foreground

**추천 아이콘 매핑**:
| 개념 | Lucide 아이콘 |
|------|--------------|
| 교차 정산 | `ArrowLeftRight` 또는 `Shuffle` |
| 최소 송금 | `TrendingDown` 또는 `Minimize2` |
| 로그인 불필요 | `UserX` 또는 `Zap` |
| 링크 공유 | `Share2` 또는 `Link` |
| 참가자 추가 | `UserPlus` |
| 결제 추가 | `PlusCircle` 또는 `Receipt` |
| 결과 확인 | `CheckCircle` 또는 `BarChart3` |
| 빠른 속도 | `Zap` 또는 `Timer` |

### 9.2 일러스트

- **스타일**: 라인 기반 미니멀 일러스트, 단색(primary) + 배경색
- **주제**: 친구들이 함께하는 상황 (식사, 여행, 카페)
- **제작 방향**:
  - 1순위: 실제 앱 스크린샷을 iPhone 목업에 넣은 이미지
  - 2순위: 간단한 SVG 일러스트 (사람 아이콘 + 화살표 + 돈)
  - CSS만으로 구현 가능한 추상 패턴도 고려 (원, 선, 점)

### 9.3 앱 목업

- iPhone 15 스타일 프레임 (둥근 모서리, 노치)
- 실제 앱 화면 스크린샷 삽입
- 배경에 그림자 + 미세 회전(3~5deg)으로 입체감
- CSS로 구현하거나, 미리 렌더링된 PNG/WebP 사용

  how-it-works.tsx

- `Button` (CTA용, lg/default 사이즈)
